# Implementation Plan

## Overview

Refactor `AgentCallbacks` into a set of functional lifecycle hooks organized by domain. Each hook is a trait with default no-op implementations, and `AgentCallbacks` becomes a composite struct holding optional hook instances. This eliminates the current monolithic 15+ callback field pattern in favor of composable, type-safe trait objects that are easier to extend, test, and reason about.

The current `AgentCallbacks` struct has 15+ individual `Option<Box<dyn Fn(...)>>` fields. The new design groups related callbacks into domain-specific traits organized by lifecycle scope:

- **AgentLoopHooks** — agent loop start/end
- **TurnLifecycleHooks** — per-turn pre/post hooks
- **ApiLifecycleHooks** — LLM API call events
- **ToolLifecycleHooks** — tool execution events
- **StreamingHooks** — LLM output streaming
- **SystemEventsHooks** — status, diagnostics, logging
- **SessionLifecycleHooks** — session rotation, compression
- **InterruptLifecycleHooks** — interrupt events
- **CLIInteractionHooks** — CLI-specific I/O hooks
- **ClarificationHooks** — user clarification requests

`TurnState` in TUI is a mutable state machine for UI rendering and should remain unchanged. `EventBus` can optionally implement hook traits for cleaner TUI integration.

## Types

### 1. AgentLoopHooks — Agent 整体循环

```rust
/// Agent loop lifecycle — fires once per run_loop invocation.
pub trait AgentLoopHooks: Send + Sync {
    /// Called when the agent loop starts (before any turns).
    fn on_loop_start(&self) {}
    /// Called when the agent loop ends (after all turns complete).
    /// `outcome` describes why the loop ended: "completed", "interrupted", "budget_exhausted", etc.
    fn on_loop_end(&self, outcome: &str) {}
}
```

**调用位置**: `ConversationLoop::run_loop` 入口/退出

### 2. TurnLifecycleHooks — 单个 Turn 生命周期

```rust
/// Per-turn lifecycle — fires once per execute_turn call.
pub trait TurnLifecycleHooks: Send + Sync {
    /// Called before the turn cycle begins (budget check, compression prep).
    fn on_pre_turn(&self) {}
    /// Called after the turn cycle ends (all tool calls resolved or final response).
    /// `response` is the final text output. `success` indicates if the turn completed without error.
    fn on_post_turn(&self, response: &str, success: bool) {}
}
```

**调用位置**: `TurnExecutor::execute_turn_with_config` 循环入口/退出

### 3. ApiLifecycleHooks — LLM API 调用

```rust
/// LLM API call lifecycle.
pub trait ApiLifecycleHooks: Send + Sync {
    /// Called before the API request is sent.
    fn on_api_call_start(&self) {}
    /// Called when the API request completes successfully.
    fn on_api_call_complete(&self) {}
    /// Called when the API request fails (after all retries exhausted).
    fn on_api_call_error(&self, error: &str) {}
}
```

**调用位置**: `turn_executor.rs` retry_with_backoff 前后

### 4. ToolLifecycleHooks — 工具执行

```rust
/// Tool execution lifecycle.
pub trait ToolLifecycleHooks: Send + Sync {
    /// Tool call generated by LLM (before execution).
    fn on_tool_gen(&self, tool_name: &str, call_id: &str) {}
    /// Tool execution begins.
    fn on_tool_start(&self, tool_name: &str, args: &str) {}
    /// Tool execution completes.
    fn on_tool_complete(&self, tool_name: &str, args: &str, result: &str) {}
    /// Tool progress update (optional, for long-running tools).
    fn on_tool_progress(&self, tool_name: &str, preview: &str) {}
}
```

**调用位置**: `turn_executor.rs` concurrent_dispatch 前后

### 5. StreamingHooks — 流式输出

```rust
/// LLM output streaming events.
pub trait StreamingHooks: Send + Sync {
    /// Main response text delta.
    fn on_stream_delta(&self, text: &str) {}
    /// Thinking/reasoning text delta (from models that support it).
    fn on_thinking(&self, text: &str) {}
    /// Reasoning text delta (separate from thinking).
    fn on_reasoning(&self, text: &str) {}
    /// Interim assistant message (non-streaming, full text).
    fn on_interim_assistant(&self, text: &str) {}
}
```

**调用位置**: `stream_processor`, `turn_executor.rs` LLM 响应处理

### 6. SystemEventsHooks — 系统状态

```rust
/// System status and diagnostic events.
pub trait SystemEventsHooks: Send + Sync {
    /// Status message with level (e.g., "lifecycle", "warn", "info").
    fn on_status(&self, level: &str, message: &str) {}
    /// Step-by-step progress message.
    fn on_step(&self, message: &str) {}
    /// Verbose print — always visible even during streaming.
    fn on_vprint(&self, message: &str) {}
}
```

**调用位置**: `turn_executor.rs` 各阶段状态更新

### 7. SessionLifecycleHooks — Session 生命周期

```rust
/// Session lifecycle events.
pub trait SessionLifecycleHooks: Send + Sync {
    /// Called when a session is rotated (compressed → parent ended, child created).
    /// `parent_id` is the old session, `child_id` is the new session.
    fn on_session_rotate(&self, parent_id: &str, child_id: &str) {}
    /// Called before context compression begins.
    fn on_compression_start(&self, message_count: usize) {}
    /// Called after context compression completes.
    /// `status` describes the result: "compacted", "unchanged", "ineffective".
    fn on_compression_complete(&self, status: &str) {}
}
```

**调用位置**: `turn_executor.rs` auto-compression 和 session rotation

### 8. InterruptLifecycleHooks — 中断事件

```rust
/// Interrupt lifecycle events.
pub trait InterruptLifecycleHooks: Send + Sync {
    /// Called when an interrupt is requested (user presses Ctrl+C).
    fn on_interrupt_requested(&self) {}
    /// Called when the turn actually responds to the interrupt.
    fn on_interrupted(&self, reason: &str) {}
}
```

**调用位置**: `turn_executor.rs` interrupt 检查点

### 9. CLIInteractionHooks — CLI 交互

```rust
/// CLI-specific interaction hooks.
pub trait CLIInteractionHooks: Send + Sync {
    fn on_print_prompt(&self) {}
    fn on_print_flush(&self) {}
    fn on_print_info(&self, message: &str) {}
    fn on_print_newline(&self) {}
    fn on_read_input(&self) -> Option<String> { None }
    fn on_should_exit(&self, input: &str) -> bool { false }
}
```

**调用位置**: `agent.rs::interactive_chat`, CLI dispatch

### 10. ClarificationHooks — 澄清请求

```rust
/// Clarification request — asks user for disambiguation.
pub trait ClarificationHooks: Send + Sync {
    fn on_clarify(&self, question: &str, choices: &[String]) -> String { String::new() }
}
```

**调用位置**: `AgentCallbacks::call_clarify`

### Adapter Types

```rust
/// Adapter that wraps individual Fn closures into ToolLifecycleHooks impl.
pub struct ToolLifecycleAdapter {
    pub gen: Option<Box<dyn Fn(&str, &str) + Send + Sync>>,
    pub start: Option<Box<dyn Fn(&str, &str) + Send + Sync>>,
    pub complete: Option<Box<dyn Fn(&str, &str, &str) + Send + Sync>>,
    pub progress: Option<Box<dyn Fn(&str, &str) + Send + Sync>>,
}
impl ToolLifecycleHooks for ToolLifecycleAdapter { ... }
```

Similar adapters for all traits for backward compatibility migration.

### Modified AgentCallbacks

```rust

pub struct AgentCallbacks {

    pub tool_lifecycle: Option<Box<dyn ToolLifecycleHooks>>,
```

## Functions

### Removed Functions (from AgentCallbacks)

All `call_*` methods removed — replaced by direct trait method calls:

- `call_tool_progress`, `call_tool_start`, `call_tool_complete`, `call_tool_gen`
- `call_stream_delta`, `call_thinking`, `call_reasoning`, `call_interim_assistant`
- `call_status`, `call_step`, `call_vprint`
- `call_clarify`

### New Functions

- Adapter builders: `ToolLifecycleAdapter::new()`, `StreamingAdapter::new()`, etc.
- New hook dispatch in `AgentCallbacks`:
  - `on_loop_start()`, `on_loop_end(outcome)`
  - `on_pre_turn()`, `on_post_turn(response, success)`
  - `on_api_call_start()`, `on_api_call_complete()`, `on_api_call_error(error)`
  - `on_session_rotate(parent_id, child_id)`
  - `on_compression_start(msg_count)`, `on_compression_complete(status)`
  - `on_interrupt_requested()`, `on_interrupted(reason)`

### turn_executor.rs Changes

```rust
// Before:
if let Some(cb) = config.callbacks {
    cb.call_status("lifecycle", "turn_start");
}

// After:
if let Some(cb) = &config.callbacks {
    if let Some(hook) = &cb.turn_lifecycle {
        hook.on_pre_turn();
    }
    if let Some(hook) = &cb.system_events {
        hook.on_status("lifecycle", "turn_start");
    }
}
```

### agent.rs Changes

```rust
// In interactive_chat:
if let Some(cb) = &self.callbacks {
    if let Some(hook) = &cb.agent_loop {
        hook.on_loop_start();
    }
}

// At end of run_loop:
if let Some(cb) = &self.callbacks {
    if let Some(hook) = &cb.agent_loop {
        hook.on_loop_end("completed");
    }
}
```

## Implementation Order

1. **Create all 10 hook traits** in `oben-agent/src/hooks/kind.rs` (with `Hooks` suffix)
2. **Create 10 adapter types** in `oben-agent/src/hooks/adapters.rs`
3. **Refactor AgentCallbacks** in `oben-agent/src/callbacks/mod.rs`
4. **Update turn_executor.rs** — all callback invocations
5. **Update conversation.rs** — TurnOptions callback usage
6. **Update agent.rs** — loop-level hooks in interactive_chat
7. **Update CLI dispatch.rs** — callback construction
8. **Update TUI lib.rs** — callback construction
9. **Run full test suite** — verify no regressions
