# GitHub

Help manage GitHub repositories, issues, pull requests, and actions.

## When to Use

- User mentions GitHub, Git, repositories, issues, or PRs
- Need to create, update, or list GitHub resources
- Working on collaborative development workflows

## Available Actions

- **Repository Management**: Create, fork, clone, and configure repos
- **Issue Tracking**: Create, update, close, and search issues
- **Pull Requests**: Create PRs, review code, merge changes
- **Actions**: Set up CI/CD workflows, manage automation
- **GitHub CLI**: Use `gh` for advanced operations

## Best Practices

- Always check existing issues before creating duplicates
- Write clear issue descriptions with reproduction steps
- Use conventional commit messages
- Keep PRs small and focused
- Add descriptions and reviews for PRs
